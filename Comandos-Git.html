Que es Git>

Es un sistema de control de verisones que sirve para alojar codigos de aplicaciones de cualquier desarrolador , esto nos refiere a una de sus funciones principales la cual es clonar los codigos , permitiendo optimizar los cambios en proyectos asi como agilizar el almacenamiento de informacion

Lista de comandos:
  1. Init
  2. Status
  3. Branch
  4. Checkout
  5. Remote
  6. Pull
  7. Push
  8. Clone
  9. Commit
    10. Init
    Crea un nuevo repositorio de Git. Puede utilizarse para convertir un proyecto existente y sin version en un repositorio de Git, o para inicializar un nuevo repositorio vacio.
    git init
    Status
    Muestra el estado del directorio de trabajo y del area del entorno de ensayo. Permite ver los cambios que se han preparado, los que no y los archivos en los que Git no va a realizar el seguimiento.
    git status
    Branch
    Una rama representa una linea independiente de desarrollo. Las ramas sirven como una abstraccion de los procesos de cambio, preparacion y confirmacion. Puedes concebirlas como una forma de solicitar un nuevo directorio de trabajo, un nuevo entorno de ensayo o un nuevo historial de proyecto.
    git branch (poner cualquier nombre), git branch (solo git branch es para ver en que rama te ubicas)
    Checkout
    Si encuentras una referencia de confirmacion al punto del historial que quieres visitar, puedes utilizar el comando git checkout para visitar dicha confirmacion. git checkout es una forma sencilla de “cargar” cualquiera de estas instantaneas guardadas en tu maquina de desarrollo
    git checkout tiene muchas funciones pero la mas normal es cambiar de rama git checkout (nombre de la rama)
    Remote
    Te permite crear, ver y eliminar conexiones con otros repositorios. Las conexiones remotas se asemejan mas a marcadores que a enlaces directos con otros repositorios. En lugar de brindar acceso en tiempo real a otro repositorio, funcionan como nombres practicos que pueden emplearse para hacer referencia a una URL no tan sencilla.
    git remote
    Pull
    Se emplea para extraer y descargar contenido desde un repositorio remoto y actualizar al instante el repositorio local para reflejar ese contenido.
    git pull
    Push
    Se usa para cargar contenido del repositorio local a un repositorio remoto. El envio es la forma de transferir confirmaciones desde tu repositorio local a un repositorio remoto.
    git push
    Clone
    Es una utilidad de linea de comandos de Git que se utiliza para fijar como objetivo un repositorio existente con el fin de clonarlo o copiarlo.
    git clone
    Commit
    Captura una instantanea de los cambios preparados en ese momento del proyecto. Las instantaneas confirmadas pueden considerarse como versiones "seguras" de un proyecto: Git no las cambiara nunca a no ser que se lo pidas expresamente.
    git commit
